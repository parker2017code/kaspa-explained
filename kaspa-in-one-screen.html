Kaspa in one screen

Proof of Work redesigned around parallel blocks.

Kaspa keeps mined security culture and a UTXO model, then uses a blockDAG and GHOSTDAG so honest blocks found in parallel can be included and ordered instead of treated as ordinary wasted side branches.

The artifact

The shortest serious version.

What it is

A Proof-of-Work UTXO blockDAG network for lower-latency shared state.

What is live

PoW mining, UTXO model, blockDAG structure, GHOSTDAG ordering, and the Crescendo-era 10 BPS network.

Why it matters

It tests whether Bitcoin-style PoW settlement can feel closer to real time without becoming a normal centralized fast database.

What is not live

Toccata, vProgs, native DeFi, DAGKnight, 100 BPS, TangVM, and RTD-derived oracle or attestation flows should not be treated as shipped mainnet functionality.

What would make it work

Durable users, healthy mining, realistic node operation, liquidity, fees, wallets, developer tools, useful post-Toccata apps, and careful roadmap execution.

What would weaken it

Overclaiming roadmap work, weak adoption, poor liquidity, mining or infrastructure concentration, unclear security budget, or app-layer promises outrunning shipped software.

Mental shortcut

The narrative spine.

Bitcoin-style Proof-of-Work security culture, seconds-scale user feel, blockDAG ordering, and future verification-oriented apps.

That is the compact public narrative. The important caveat is that the last part is future-facing and must stay status-labeled.
