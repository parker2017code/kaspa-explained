Parallel blocks, one history

Kaspa is real-time Proof of Work.

It keeps mined Proof of Work and a UTXO model, then uses a blockDAG and GHOSTDAG so honest blocks can be found in parallel and still ordered into one shared history.

Kaspa's core move is simple: do not force honest work to wait in a single-file chain when it can stay in the graph and still be ordered into one shared payment history.

Common mistakes

Speed claims need context.

Real-time Proof of Work is a useful frame, but it should not be stretched into every exciting number people repeat online.

Block rate

10 BPS is live. That does not automatically mean 10x finality, unlimited throughput, or every lab/demo number as normal mainnet use.

TPS

Capacity tests, average sustained capacity, policy limits, fees, and organic demand are different measurements. Say which one you mean.

Testnets

Testnet experiments can point toward future capability. They are not the same as current mainnet guarantees.

Roadmap

DAGKnight, vProgs, native DeFi, TangVM, and RTD-derived oracle flows stay in later lanes until primary activation evidence changes the status.

Why it is interesting

Why it is interesting.

PoW, but lower latency

Kaspa keeps mined work and tries to make the user experience feel closer to real-time payments instead of waiting for a slow single-chain cadence.

Parallel honest work

The blockDAG shape lets honest blocks found at nearly the same time remain part of the history, then GHOSTDAG orders that graph.

Future app rails

The roadmap points toward apps that prove their own rules while the base layer stays focused on ordering payments and commitments.

Honest uncertainty

The open question is whether this architecture can attract durable usage, liquidity, fees, and builders at production scale.

Apps matter

What can people build on it?

Kaspa's app story is not "every roadmap idea is live." It is a staged path: useful wallets and payment UX now, vault and asset rules after Toccata, and richer apps that prove their rules later if the roadmap ships.

Now

Wallets, payments, merchant flows, dashboards, mining/node visibility, and clearer confirmation-risk UX.

Toccata lane

Vault policies, simple assets, escrow, assurance contracts, and bounded UTXO state machines.

Later

Apps that can run markets, lending, funding rules, attestations, and other shared commitments if the roadmap ships and users show up.

One visual idea

Bitcoin chooses one path. Kaspa orders parallel work.

A normal blockchain has one selected chain. When honest miners find blocks at nearly the same time, one path wins and the other competing blocks usually become wasted side blocks. Kaspa's blockDAG starts from a different shape: honest parallel blocks can reference one another and GHOSTDAG gives nodes a rule for ordering that graph.
